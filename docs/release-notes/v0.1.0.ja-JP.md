# Hermes Task Board v0.1.0

**初回パブリックリリース** — 2026-04-19

[Hermes Agent](https://github.com/NousResearch/hermes-agent) をバッチ駆動する Trello 風かんばん。単一 Go バイナリ、SQLite + ファイルシステムストレージ、Vue 3 フロントエンド（ビルドステップ不要）、バイリンガル UI（英語 / 简体中文）。

## このリリースに含まれるもの

### ボード & タスクワークフロー
- 4 カラムかんばん: Draft → Plan → Execute → Verify → Archive、カラム内でのドラッグ並べ替えとカラム間ドラッグ遷移に対応。
- すべてのカードに優先度バッジ（P0–P3）とカラータグチップを表示。
- タスク依存関係: Plan 内のタスクが依存する他のタスクをリストでき、スケジューラは依存関係が要求された状態に到達するまで Execute への移動を待機。
- 直列または並列ディスパッチ: スケジューラはサーバーレベルおよびプロファイルレベルの同時実行数上限を監視し、収まるもののみを実行。

### Hermes 統合
- プラグ可能な `hermes_servers` リスト — 任意の数の Hermes ゲートウェイインスタンスを登録可能。それぞれ独自の API キー（保管時に暗号化）、モデル、同時実行数上限を持つ。
- サーバーごとの上限に加えて、プロファイルごとの同時実行数制限。
- すべてのタスクディスパッチは **Attempt** を作成: Hermes とのライブ会話。ストリーム出力は NDJSON として永続化され、SSE 経由でブラウザにブロードキャスト。
- カード内チャット: フォローアップメッセージを送信して Attempt を継続（**Ctrl/⌘ + Enter** で送信）。
- 実行中の Attempt を 2 クリックで**停止** — 誤ったキャンセルを防止。

### 定期タスク
- フレンドリーなスケジュールピッカーとプリセットモード:
  - 毎 N 分 / 毎 N 時間（N は自由形式の数値）
  - 毎日 HH:MM
  - 毎週 選択した曜日の HH:MM
  - 毎月 指定日の HH:MM
  - 上級者向け 高度な設定（生 5 フィールド cron）
- ピッカーは保存される cron 式のライブプレビューを表示し、保存済みのスケジュールを人間にわかりやすい文章で表示（「每 15 分钟」「Weekly Mon, Wed, Fri at 09:00」）。
- バックエンドは標準 5 フィールド cron のみ保存（robfig/cron/v3）。プレリリース DB に残るレガシー `interval` 行は初回起動時に cron に書き換えられる。ワーカーは起動時に欠落している `next_run_at` タイムスタンプも復元。

### タグシステムプロンプト
- すべてのタグにオプションの **システムプロンプト** を設定可能。タスクがディスパッチされると、そのタスクのすべてのタグのプロンプトが Hermes に渡される基本ペルソナに連結される。例えば `notify-qq` タグに「post a short summary to QQ when done」というプロンプトを設定すれば、そのタスクは完了時に自動的に QQ に短い要約を投稿する。

### 信頼性
- **孤児 Attempt 回収**: 起動時に `queued` または `running` のままの Attempt（クラッシュや強制再起動の残骸）をシステムイベント付きで `failed` に反転。これにより UI が生きた Attempt で永遠にスピナーを表示し続けたり、同時実行スロットがリークしたりするのを防ぐ。
- **夜間ファイルシステム回収**: 90 日以上経過し、一致する DB 行を持たない `data/attempt/*` ディレクトリを削除。

### UX ポリッシュ
- 設定ドロップダウンでホットスワップするバイリンガル UI（英語 / 简体中文）。
- ダーク / ライトテーマ切替。
- アクティブなカードに注目を集めるエレクトリックチェイスボーダー:
  - 緑+赤 — Attempt がアクティブに実行中
  - オレンジ+赤 — Attempt がユーザー入力を必要とする（またはタスクが Verify で待機中）
- チャット風イベントストリーム: 下部にピン留め中は自動スクロール。履歴を読むために上にスクロールすると、下に引きずり込むのを止め、「↓ 新着メッセージ」ピルを表示。
- カードのドラッグとクリックが明確に区別 — ドラッグしてもカードモーダルが開かなくなった。
- 画像アップロードは Aliyun OSS 設定の背後で制限（ローカルサーバーは Hermes が実際にアクセスできる画像をホストできないため）。

## クイックスタート

```bash
tar -xzf hermes-taskboard-v0.1.0-linux-amd64.tar.gz
cd hermes-taskboard-v0.1.0-linux-amd64
# 初回起動時に data/config.yaml がデフォルト値で作成される。設定をカスタマイズするには
# config.example.yaml からコピー（Hermes の base_url、api_key、モデル、同時実行数上限）。
./hermes-taskboard -data ./data
# → http://127.0.0.1:1900 をブラウザで開く
```

Docker イメージも公開されています。リポジトリ内の `Dockerfile` を参照してください。

## プレリリース main からのアップグレード

このタグの前に `main` からのビルドを実行していた場合、新しいバイナリに置き換えて再起動するだけで済みます。すべてのスキーママイグレーションは冪等です。初回起動時に新しいバイナリは以下を実行します:

1. `task_schedules` 内のレガシー `kind='interval'` 行を同等の cron 式に書き換え（例: `15m` → `*/15 * * * *`）、`next_run_at` をクリア。
2. 有効で NULL タイムスタンプを持つスケジュールの `next_run_at` を再計算。
3. 旧プロセスが残した `queued` / `running` Attempt を `failed` にマーク（起動ログに `reaped orphan attempts count=N` が表示されます）。

手動マイグレーションは不要です。

## 既知の制限

- タスクボード再起動後に Hermes 接続は再開されません。シャットダウン時にストリーム中だった Attempt は failed としてマークされます。会話履歴はディスク上に保存されています。続行するには新しいメッセージを送信してください。
- デプロイメントごとに認証ユーザーは 1 名のみ（または認証無効）。マルチユーザー ACL はこのリリースには含まれていません。
